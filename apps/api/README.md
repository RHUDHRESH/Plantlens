# apps/api — the FastAPI modular monolith (the spine)

One process, internal modules. This is where the product's brain lives: ingest → alarm → DAG →
situation → calm card, plus the Studio compiler, incidents, auth, and the audit chain. Read
`docs/ARCHITECTURE.md` for the runtime tick and `docs/BUILD_ORDER.md` for the sequence.

## Top-level files
| File | Purpose | Chunk |
|------|---------|-------|
| `pyproject.toml` | deps (per chunk), tooling config | 0 |
| `alembic.ini` | Alembic migration config | 1 |
| `.env.example` | documented env vars (copy to `.env`) | 1 |
| `migrations/` | Alembic versions (autogenerated) | 1 |
| `app/` | the application package (see `app/README.md`) | 1+ |
| `tests/` | unit / integration / contract / golden / scenario tests | all |
| `compiled/` | compiler output (gitignored; regenerated) | 4 |

## The module map (under `app/`)
| Module | Responsibility | Chunk | Differentiator? |
|--------|----------------|-------|-----------------|
| `core` / top-level | `main`, `lifespan`, `settings`, `dependencies` | 1 | — |
| `db` | SQLAlchemy models + repositories (authored vs derived kept separate) | 1 | — |
| `auth` | OIDC/JWT verify + RBAC (viewer/engineer/operator/maintenance/admin/agent) | 1 | — |
| `services` | `audit_chain` (hash-chain), `export_service` | 1 | audit = "receipts" |
| `schemas` | Pydantic transport models mirroring `packages/contracts` | 1+ | — |
| `routers` | thin HTTP/WS layer; one file per resource | 1+ | — |
| `runtime` | **alarm + DAG + situation + calm-card + asset-status + projection** | 3 | **YES — the core** |
| `runtime/simulator` | scenario runner + simulator gateway (emits TagFrame) | 2 | enables everything |
| `studio` | the compiler: contracts → compiled_hmi.json (+ validators) | 4 | "matrix compiles UI" |
| `ingest` | Python port of the cliffords pipeline (CSV/Excel/OPC-UA → canonical) | later | — |
| `incidents` | incident room service/store/timeline/checklist | 10 | response workflow |

## Run
```bash
uv sync && uv run uvicorn app.main:app --reload   # dev
```

## The one rule that keeps this clean
`schemas/` (Pydantic transport) and `db/models/` (SQLAlchemy persistence) are **different layers**.
Never make one class serve both. See `app/schemas/README.md` and `app/db/README.md`.
