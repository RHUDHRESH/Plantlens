`demo-microgrid/` is the canonical authored bundle for the hero demo (see `docs/DEMO_SCENARIO.md`).
Every file here validates against a contract in `packages/contracts/`. This bundle is:

- the **smoke test** for the whole contract layer (a CI job validates it),
- the **fixture** the simulator and runtime build against,
- the **seed** Studio loads so you don't start from an empty canvas.

## What is NOT here
The **compiled** outputs (`compiled_hmi.json`, runtime indexes) are generated by the Studio
compiler (Chunk 4). They land in `apps/api/compiled/` (or object storage), never checked in here,
never hand-edited. To regenerate: `POST /api/v1/compiler/compile`.

## Consistency invariants (the compiler validators enforce these)
- Every `tag_map.tags[].asset_id` exists in `plant.json`.
- Every `alarm_rules.rules[].tag` exists in `tag_map.json`.
- Every `causal_graph.edges[].{from,to}` exists as a node.
- Every `scenarios[].events[].tag` exists in `tag_map.json`.
- The causal graph is acyclic.
- `scn_motor_overload` must yield exactly one Situation rooted at `MTR-301`.
