# packages/contracts — the schema spine (single source of truth)

Every service compiles from these JSON Schemas. Read `docs/CONTRACTS.md` for the philosophy.

## Sub-folders
- `openapi/` — the API's exported `openapi.json` (generated by the backend; used to generate the
  web client's TS types). Empty until Chunk 1 runs and you export it.
- `generated-ts/` — TS types generated from the schemas/OpenAPI (build artifact; gitignored).

## The discipline
- Backend Pydantic (`apps/api/app/schemas/`) mirrors these; CI diffs `model_json_schema()` output
  against these files (`contract-validate` job). Drift fails the build.
- Frontend Zod (`apps/web/src/app/schemas/`) mirrors these.
- The demo bundle in `packages/sample-data/demo-microgrid/` MUST validate against all of them —
  this is the smoke test for the whole contract layer.
